## Overview
Welcome to the revamped **DX Innovation Hub**. This project is a complete high-fidelity rebrand of the original **DXHub Innovations** website. Our goal was to elevate the digital experience to an **S+ Grade** professional aesthetic, drawing inspiration from Apple’s minimalist and premium design language.

We have moved away from generic UI patterns to a custom-engineered, performance-optimized interface that scales seamlessly across all devices.

## Design Philosophy
The new DXHub is built on three core pillars:
1. **Premium Minimalism**: A clean, spacious layout using the **Deep Blue & Canary Yellow** palette to convey trust and innovation.
2. **Grid-Less Architecture**: We have eliminated dependencies on standard CSS/MUI grids in favor of custom **Flexbox** and **CSS Grid** (`Box` based) layouts for maximum control and responsiveness.
3. **Dynamic Motion**: Leveraging **Framer Motion** for smooth, non-intrusive entrance animations and interactive micro-states that make the site feel "alive."

## Key Features
- **High-Fidelity Sub-pages**: Fully functional pages for Products, Solutions, Resources, Pricing, and Contact.
- **Micro-Bootcamp & WorkLab**: Dedicated sections and pages for our core innovation ecosystems.
- **Scroll-to-Top Logic**: Intelligent navigation that ensures a fresh start whenever you switch pages.
- **Mobile Optimized**: Every component is hand-tuned for mobile, with intelligent image scaling and centering.
- **SEO & Performance**: Optimized meta tags, semantic HTML, and high-performance production builds.

## Tech Stack
- **Framework**: [React](https://reactjs.org/) + [Vite](https://vitejs.dev/)
- **UI Architecture**: [Material UI (MUI)](https://mui.com/) (Custom Styling, Grid-free)
- **Animations**: [Framer Motion](https://www.framer.com/motion/)
- **Routing**: [React Router 6](https://reactrouter.com/)
- **Styling**: Vanilla CSS + MUI Emotion

## Getting Started
To run the project locally:

1. **Clone the repository**
2. **Install dependencies**:
   ```bash
   npm install
   ```
3. **Run the dev server**:
   ```bash
   npm run dev
   ```
4. **Build for production**:
   ```bash
   npm run build
   ```
